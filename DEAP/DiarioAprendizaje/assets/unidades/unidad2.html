HTTP & DNS

Introducción a HTTP

¿Que es HTTP?

(Hypertext Transfer Protocol) es un protocolo de comunicación que se utiliza para transferir información en la web. Es la base de cualquier intercambio de datos en la web y un protocolo de comunicación de cliente-servidor.

El navegador - Cliente Web

Puede Negociar

Puede negociar algunas caracteristicas particulares. Por ejemplo, si se soporta compresión o no.

Interpreta Informacion

Interpreta la información en los documentos y renderiza el documento.

Recogida de Información

Recoge información del usuario y la envía al servidor como parámetros en una petición web GET o POST.

Gestión de Cookies

Gestiona las cookies que han sido enviadas por el servidor.

El servidor web

El servidor web es un programa que se ejecuta en un ordenador y que se encarga de servir documentos HTML a los clientes web. El servidor web recibe las peticiones de los clientes web y envía los documentos HTML solicitados.

El protocolo HTTP

Es orientado a peticion / respuesta. El cliente envía una petición al servidor y el servidor responde con un mensaje de respuesta. El mensaje de respuesta contiene información sobre el estado de la petición y el documento solicitado.

Métodos que soporta

GET

Cuando se envía una petición GET, los datos que se envían al servidor se incluyen en la URL de la solicitud.

POST

El método POST se utiliza para enviar datos al servidor en el cuerpo de la solicitud, no en la URL.

HEAD

El método HEAD solicita información de un recurso sin descargar su contenido. El servidor responde con los encabezados HTTP del recurso.

PUT

El método PUT se utiliza para enviar datos al servidor para crear o actualizar un recurso.

DELETE

El método DELETE se utiliza para eliminar un recurso del servidor.

CONNECT

El método CONNECT se utiliza para establecer una conexión con el servidor.

OPTIONS

El método OPTIONS se utiliza para obtener información sobre los métodos HTTP que un servidor soporta.

TARCE

El método TRACE se utiliza para realizar una prueba de bucle de retorno de un mensaje de solicitud.

PACTH

El método PATCH se utiliza para aplicar modificaciones parciales a un recurso.

El código de estado HTTP

El código de estado HTTP es un número que se devuelve en una respuesta HTTP para indicar el resultado de la solicitud. Los códigos de estado HTTP se dividen en cinco clases. Cada clase tiene un rango de códigos de estado.

Clases de códigos de estado HTTP

1XX

Respuestas informativas. Indican que la solicitud ha sido recibida y el servidor está procesando la solicitud.

2XX

Respuestas de éxito. Indican que la solicitud fue recibida, entendida y aceptada.

3XX

Redirecciones. Indican que el cliente necesita realizar una acción adicional para completar la solicitud.

4XX

Errores del cliente. Indican que la solicitud contiene sintaxis incorrecta o no se puede cumplir.

5XX

Errores del servidor. Indican que el servidor no pudo cumplir una solicitud aparentemente válida.


Introducción a DNS

(Domain Name System) es un sistema de nomenclatura jerárquico para computadoras, servicios o cualquier recurso conectado a Internet o a una red privada. Traduce nombres de dominio significativos para los humanos en direcciones IP numéricas asociadas con los equipos de servicios.

¿Cómo funciona?

Cuando escribes una dirección web en tu navegador, el navegador envía una solicitud al servidor DNS para traducir el nombre de dominio en una dirección IP. El servidor DNS busca la dirección IP asociada con el nombre de dominio y la devuelve al navegador.

¿Qué es un nombre de dominio?

Un nombre de dominio es una cadena de caracteres que se utiliza para identificar un recurso en Internet. Los nombres de dominio se utilizan en lugar de las direcciones IP numéricas para identificar los recursos en la web.

Sistema de Nombres de Dominio

1 PASO

Se introduce la dirección web en el navegador.

2 PASO

Se solicita al ISP la dirección IP del servidor DNS.

3 PASO

Se solicita la resolucion del nombre de dominio al servidor DNS.

4 PASO

Se retorna con la dirección IP asociada con el nombre de dominio.

5 PASO

El cliente se conecta al servidor web utilizando la dirección IP.

Tipos de Registros

A

El registro A se utiliza para asociar un nombre de dominio con una dirección IP.

AAAA

El registro AAAA se utiliza para asociar un nombre de dominio con una dirección IPv6.

CNAME

El registro CNAME se utiliza para asociar un nombre de dominio con otro nombre de dominio.

NS

El registro NS se utiliza para asociar un nombre de dominio con un servidor de nombres.

MX

El registro MX se utiliza para asociar un nombre de dominio con un servidor de correo.

PTR

El registro PTR se utiliza para asociar una dirección IP con un nombre de dominio.

SOA

El registro SOA se utiliza para almacenar información sobre la zona de autoridad.

Resolución directa vs Resolución Inversa

La resolución directa se utiliza para traducir un nombre de dominio en una dirección IP. La resolución inversa se utiliza para traducir una dirección IP en un nombre de dominio.

La resolución inversa se utiliza para traducir una dirección IP en un nombre de dominio. La resolución inversa se utiliza para traducir una dirección IP en un nombre de dominio.

Herramientas DNS de consulta

Hay varias herramientas que se pueden utilizar para consultar un servidor DNS. Algunas de las herramientas más comunes son nslookup, dig y host.
