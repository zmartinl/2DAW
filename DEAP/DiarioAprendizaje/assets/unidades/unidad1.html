GIT

Introducción a GIT

¿Que es GIT?

Git es un SERVICIO (estándar) de comunicación y gestión de información entre local y remoto. Es un sistema de control de versiones distribuido, gratuito y de código abierto.

Registro los cambios en un archivo o proyecto, definiendo diferentes versiones, que pudieran recuperarse de manera específica.

Varios IDEs (Entornos de Desarrollo Integrados) como Visual Studio Code, Atom, Sublime Text, entre otros, tienen integración con Git, lo que facilita el uso de este sistema de control de versiones.

Control de versiones

Centralizado

En un sistema de control de versiones centralizado, los archivos se almacenan en un servidor central. Los usuarios obtienen una copia de los archivos y los modifican en su computadora local. Cuando los usuarios desean compartir sus cambios, los envían al servidor central.

Distribuido

En un sistema de control de versiones distribuido, cada usuario tiene una copia local de los archivos y el historial de cambios. Los usuarios pueden trabajar de forma independiente y enviar sus cambios a otros usuarios. Los cambios se pueden fusionar de forma automática.

Areas

Directorio de Trabajo

Es el directorio de trabajo, donde se encuentran los archivos que se están editando.

Staging Area

Es el área de preparación, donde se encuentran los archivos que se van a enviar al repositorio.

Repositorio Local

Es el repositorio que se encuentra en local. Aquí es donde se almacenan los cambios confirmados antes de ser enviados a un repositorio remoto.

Repositorio Remoto

Aquí es donde se almacenan los cambios confirmados que se han enviado desde el repositorio local.

Los 5 estados de GIT ( ficheros )

Untracked

El fichero no está bajo el control de versiones. No está en el repositorio.

Unmodified

El fichero está en el repositorio pero no ha sido modificado desde su última versión confirmada.

Modified

los datos/ficheros se han modifcado, pero todavía no lo has confirmado a tu base de datos,

Staged

se ha marcado un archivo modificado en su versión actual para que vaya en tu próxima confirmación.

Commited

los datos/ficheros están almacenados de manera segura en tu base de datos local.
