GIT

Introducción a GIT

¿Que es GIT?

Git es un SERVICIO (estándar) de comunicación y gestión de información entre local y remoto. Es un sistema de control de versiones distribuido, gratuito y de código abierto.

Registro los cambios en un archivo o proyecto, definiendo diferentes versiones, que pudieran recuperarse de manera específica.

Varios IDEs (Entornos de Desarrollo Integrados) como Visual Studio Code, Atom, Sublime Text, entre otros, tienen integración con Git, lo que facilita el uso de este sistema de control de versiones.

Control de versiones

Centralizado

En un sistema de control de versiones centralizado, los archivos se almacenan en un servidor central. Los usuarios obtienen una copia de los archivos y los modifican en su computadora local. Cuando los usuarios desean compartir sus cambios, los envían al servidor central.

Distribuido

En un sistema de control de versiones distribuido, cada usuario tiene una copia local de los archivos y el historial de cambios. Los usuarios pueden trabajar de forma independiente y enviar sus cambios a otros usuarios. Los cambios se pueden fusionar de forma automática.

Areas

Directorio de Trabajo

Es el directorio de trabajo, donde se encuentran los archivos que se están editando.

Staging Area

Es el área de preparación, donde se encuentran los archivos que se van a enviar al repositorio.

Repositorio Local

Es el repositorio que se encuentra en local. Aquí es donde se almacenan los cambios confirmados antes de ser enviados a un repositorio remoto.

Repositorio Remoto

Aquí es donde se almacenan los cambios confirmados que se han enviado desde el repositorio local.

Los 5 estados de GIT ( ficheros )

Untracked

El fichero no está bajo el control de versiones. No está en el repositorio.

Unmodified

El fichero está en el repositorio pero no ha sido modificado desde su última versión confirmada.

Modified

los datos/ficheros se han modifcado, pero todavía no lo has confirmado a tu base de datos,

Staged

se ha marcado un archivo modificado en su versión actual para que vaya en tu próxima confirmación.

Commited

los datos/ficheros están almacenados de manera segura en tu base de datos local.


Configuración Inicial

git --version /* Te dice la version actual de git */

git config --list /* Muestra la configuración actual de git */

git config --global user.name "Tu nombre" /* Configura el nombre de usuario */
git config --global user.email "Tu correo" /* Configura el correo de usuario */

Comandos Básicos

git help <comando> /* Muestra la ayuda del comando que se le indique. */

git init /* Inicializa un repositorio git */

Subir Cambios

git add <archivo> /* Añade un archivo al área de preparación */
git add . /* Añade todos los archivos al área de preparación */
git add .html /* Añade todos los archivos con extensión .html al área de preparación */

git commit -m "Mensaje" /* Confirma los cambios con un mensaje */
git commit --amend -m "Mensaje" /* Agrega cambios al ultimo commit */

git push /* Sube los cambios al repositorio remoto */ git push -u origin master /* Sube los cambios al repositorio remoto */

Empezando a funcionar

git status /* Muestra el estado actual del repositorio */

git log /* Historial de commits extendido */
git log --oneline /* Historial de commits en una sola linea */

git diff /* Muestra las diferencias entre el directorio de trabajo y el área de preparación */
git diff --staged /* Muestra las diferencias entre el área de preparación y el último commit */

git remote /* Muestra una lista con los nombre remotos que hayas especificado */
git remote -v /* Muestra la URL de los repositorios remotos */
git remote add origin <url> /* Añade un repositorio remoto */
git remote remove origin /* Elimina un repositorio remoto */
git remote show origin /* Muestra información detallada de un repositorio remoto */
git remote rename origin nuevoNombre /* Renombra un repositorio remoto */

git branch -M main /* Cambia el nombre de la rama master a main */
git branch /* Muestra las ramas locales */
git branch -a /* Muestra todas las ramas */
git branch -d <nombreRama> /* Elimina una rama */
git branch -m <nombreRama> /* Renombra una rama */

Rollback

git checkout <nombreRama> /* Cambia a la rama especificada */
git checkout -b <nombreRama> /* Crea una rama y cambia a ella */

git reset HEAD <archivo> /* Quita los archivos de staged y los devuelve a estado anterior */
git reset --soft <hash> /* Mueve el HEAD al commit especificado */
git reset --hard <hash> /* Mueve el HEAD al commit especificado y borra los cambios en el directorio de trabajo */
git reset --mixed <hash> /* Mueve el HEAD al commit especificado y borra los cambios en el área de preparación */

git revert <hash> /* Crea un nuevo commit que revierte los cambios de un commit especificado */
git revert HEAD /* Crea un nuevo commit que revierte los cambios del último commit */
git revert HEAD~2 /* Crea un nuevo commit que revierte los cambios de los dos últimos commits */

Etiquetas

git tag /* Muestra las etiquetas */
git tag -a v1.0 -m "Versión 1.0" /* Crea una etiqueta anotada */
git tag -d v1.0 /* Elimina una etiqueta */
