APACHE

Es un servidor web HTTP de código abierto, para plataformas Unix (BSD, GNU/Linux, etc.), Microsoft Windows, Macintosh y otras, que implementa el protocolo HTTP/1.1 y la noción de sitio virtual. Cuando se instala, se crea un servidor web que se ejecuta en segundo plano, esperando peticiones de los clientes. Cuando llega una petición, el servidor la satisface devolviendo al cliente el contenido solicitado.

Configuración

La configuración de Apache se realiza mediante archivos de texto. El archivo principal de configuración es httpd.conf, que se encuentra en el directorio de instalación de Apache. En este archivo se definen los parámetros de configuración del servidor, como el puerto en el que escucha, el número máximo de conexiones simultáneas, el directorio raíz de los documentos, etc.

Virtual Hosts

Un servidor virtual es un servidor web que comparte el mismo servidor físico con otros servidores virtuales. En un servidor virtual, se pueden alojar varios sitios web, cada uno con su propio nombre de dominio. Para configurar un servidor virtual en Apache, se debe añadir una sección al archivo de configuración httpd.conf con la siguiente estructura:

                    
                        <VirtualHost *:80>
                            ServerName www.ejemplo.com
                            DocumentRoot /var/www/ejemplo
                        </VirtualHost>

En este ejemplo, se define un servidor virtual que escucha en el puerto 80 y responde a las peticiones de www.ejemplo.com. Los documentos del sitio se encuentran en el directorio /var/www/ejemplo.

Modulos

Apache es un servidor modular, lo que significa que se puede extender con módulos que añaden funcionalidades adicionales. Algunos módulos que se pueden añadir a Apache son:

MOD_SSL

Añade soporte para SSL/TLS

MOD_PHP

Permite ejecutar scripts PHP

MOD_REWRITE

Permite reescribir las URLs de las peticiones

MOD_PROXY

Añade soporte para servidores proxy

Logs

Apache genera varios archivos de log en los que registra información sobre las peticiones que recibe y los errores que se producen. Algunos de los archivos de log más importantes son:

access_log

Registra información sobre las peticiones que recibe el servidor.

error_log

Registra los errores que se producen en el servidor.

ssl_access_log

Registra información sobre las peticiones HTTPS.

ssl_error_log

Registra los errores HTTPS.

Seguridad

Apache es un servidor web muy seguro, pero es importante configurarlo correctamente para evitar problemas de seguridad. Algunas recomendaciones para mejorar la seguridad de Apache son:

Actualizar Apache y los módulos regularmente.

Configurar correctamente los permisos de los archivos y directorios.

Utilizar SSL/TLS para cifrar las comunicaciones.

Proteger los archivos de configuración.
